I started my SAP internship knowing zero Spring Boot. Two weeks in, I had a working REST API talking to Postgres. Here's the path that worked — written so a past version of me could follow it.

## Start with one annotation at a time

The first wall I hit was the alphabet soup: `@Component`, `@Service`, `@Repository`, `@RestController`, `@Autowired`, `@Bean`. Every tutorial throws all six at you in the first 10 minutes.

The mental model that finally stuck: **`@Component` is the parent**. Spring scans your classes, sees that annotation, and remembers "I need to construct one of these and hand it out when asked."

Everything else is a flavor of `@Component`:

- `@Service` — business logic
- `@Repository` — talks to the database
- `@RestController` — handles HTTP

They are functionally identical to `@Component`. The difference is **intent** — and Spring uses the specific ones to apply default behavior (e.g. `@Repository` translates JPA exceptions for you).

## Constructor injection > field injection

Old tutorials show this:

```java
@Service
public class UserService {
  @Autowired
  private UserRepository repo;
}
```

This works, but it makes testing painful and hides dependencies. Use this instead:

```java
@Service
public class UserService {
  private final UserRepository repo;

  public UserService(UserRepository repo) {
    this.repo = repo;
  }
}
```

Spring 4.3+ injects automatically when there's only one constructor — no `@Autowired` needed. Bonus: `final` makes the field immutable and forces you to think about dependencies upfront.

## Transactions are not magic

`@Transactional` looks easy: slap it on a method, get atomic database operations. But the gotchas are real.

> **Warning:** `@Transactional` only works on **public methods called from outside the class**. Internal calls bypass the proxy entirely.

This bit me on day 5. I had a `processOrder()` method calling `saveOrder()` in the same class — both `@Transactional`. The inner one silently did nothing.

The fix: split into two services, or accept that `@Transactional` is a "between-classes" boundary, not a "between-methods" one.

## What I'd skip on a second pass

If I were doing this again, I'd skip:

- **XML config**. It's dead. Use `application.yml` and Java config.
- **Field injection tutorials**. Constructor only.
- **`spring.io/guides`** as a starting point. They assume too much. Start with [Baeldung's Spring Boot tutorial](https://www.baeldung.com/spring-boot-start) instead.

Two weeks isn't enough to be good at Spring. But it's enough to ship something real, and that's the only milestone that matters when you're learning.
