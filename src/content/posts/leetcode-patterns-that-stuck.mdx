I solved ~200 problems before any of it felt useful. Then one week something clicked, and now most "Mediums" feel like variations of four patterns. These are them.

## 1. Sliding window — when the answer is a contiguous subarray

The trigger phrase: *"longest / shortest / max sum subarray with property X"*.

Two pointers, both moving forward. The right pointer expands the window; the left pointer contracts it when a constraint is violated.

```python
def longest_unique_substring(s: str) -> int:
    seen = {}
    left = best = 0
    for right, c in enumerate(s):
        if c in seen and seen[c] >= left:
            left = seen[c] + 1
        seen[c] = right
        best = max(best, right - left + 1)
    return best
```

The pattern is so consistent that once you recognize it, the implementation falls out in 5 minutes.

## 2. BFS on grids — when "shortest path" appears

Any time the problem says "shortest", "minimum steps", or "fewest moves" on a grid or graph, it's almost always BFS. DFS finds *any* path; BFS finds the shortest one.

```python
from collections import deque

def shortest_path(grid, start, end):
    queue = deque([(start, 0)])
    seen = {start}
    while queue:
        (r, c), dist = queue.popleft()
        if (r, c) == end:
            return dist
        for dr, dc in [(-1,0), (1,0), (0,-1), (0,1)]:
            nr, nc = r+dr, c+dc
            if 0 <= nr < len(grid) and 0 <= nc < len(grid[0]) and (nr,nc) not in seen and grid[nr][nc] != '#':
                seen.add((nr, nc))
                queue.append(((nr, nc), dist+1))
    return -1
```

## 3. Tree DP — recursion that returns "info from below"

This is the one I struggled with most. The trick: each recursive call **returns information up the tree** — not just a yes/no but a structured tuple.

For "diameter of binary tree":

```python
def diameter(root):
    self.best = 0

    def depth(node):
        if not node: return 0
        left = depth(node.left)
        right = depth(node.right)
        self.best = max(self.best, left + right)
        return 1 + max(left, right)

    depth(root)
    return self.best
```

The recursion returns *depth* but updates *diameter* via a side effect. That double-tracking is the pattern.

## 4. Binary search on the answer — when it's not literally about searching

This one took me 100+ problems to internalize. If a problem asks *"what's the smallest X such that Y(X) is true?"* and `Y` is monotonic, binary search **on X**, not on the array.

Example: "ship packages in D days, what's the minimum capacity?"

You're not searching the weights array. You're searching `[max(weights), sum(weights)]` for the smallest capacity that works.

```python
def min_capacity(weights, days):
    def can_ship(cap):
        d, cur = 1, 0
        for w in weights:
            if cur + w > cap:
                d += 1
                cur = 0
            cur += w
        return d <= days

    lo, hi = max(weights), sum(weights)
    while lo < hi:
        mid = (lo + hi) // 2
        if can_ship(mid):
            hi = mid
        else:
            lo = mid + 1
    return lo
```

## Everything else?

Honestly: noise. Segment trees, advanced DP, suffix arrays — they show up in maybe 3% of interviews and you can read about them in 30 minutes if needed. The four patterns above cover ~80% of what I see now.

200 problems was a lot. If I were doing it again, I'd do 50 problems but force myself to **explain each one to a rubber duck before moving on**. That's where the patterns surface — not in the solving, but in the explaining.
